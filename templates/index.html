Technical Reference

Architecture & Engine

ColdLink Nexus is a full-stack thermodynamic freight routing platform. Every simulation runs a physically calibrated heat-transfer model against live weather and road data.

Core Stack
Python Flask
Backend API Layer
Flask REST API handles authentication (OTP + Google OAuth), receives route data from the frontend, and orchestrates the thermodynamic simulation pipeline.
Physics Engine Python
CRYO ROUTE Thermodynamic Engine
Industry-calibrated reefer truck model (R-30 insulation, 1.2 °C/min active compressor). Accounts for solar loading, ambient temperature, and traffic zone heat accumulation. Outputs a 0–100% thermal risk score per route.
JavaScript Maps API
Google Maps Integration
Directions API provides real road distance and travel duration. Places Autocomplete enables fuzzy terminal search. A custom Midnight Blue map style is applied at init.
OpenWeatherMap
Live Ambient Data
Real-time temperature and wind data fetched per route segment. Used as inputs to the thermodynamic model — no mocked weather values.
